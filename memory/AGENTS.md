## 🔖 知识使用说明

### 🔍 如何匹配关键词

**关键词匹配规则：**

1. **精确匹配**
   - 任务中出现表格"关键词"列中的任何词汇
   - 包括：技术名称、类名、方法名、注解名、配置项等
   - 示例：`Diamond.getConfig`、`@HSFProvider`、`MetaProducer`

2. **场景匹配**
   - 任务描述符合表格"使用场景"列的场景
   - 示例：需要"动态配置"→匹配Diamond、需要"权限验证"→匹配ACL

3. **模糊匹配**
   - 任务意图与某个技术栈相关
   - 示例：需要"缓存数据"→匹配Tair、需要"发送消息"→匹配MetaQ

### ⚡ 知识优先级规则

**严格遵循两级优先级：**

```
应用知识(第一优先级) > 公共知识(第二优先级)
```

**执行步骤：**

1. **第一步：检查应用知识**
   - 在"应用知识"表格中搜索是否有匹配的关键词
   - 如果找到匹配项 → **必须读取应用知识对应的文件**
   - 如果找到匹配项 → **禁止再读取公共知识的文件**

2. **第二步：降级到公共知识**
   - 仅当应用知识**完全没有匹配**时
   - 才在"公共知识"表格中搜索匹配的关键词
   - 找到匹配项后读取公共知识对应的文件

3. **禁止混用**
   - ⚠️ **同一个关键词不能同时读取应用知识和公共知识**
   - ⚠️ **应用知识优先，命中后公共知识失效**
   - ⚠️ **不同关键词可以分别在不同优先级查找**

### 📋 读取文件的方式

**读取完整文件内容：**

1. **使用Read工具**
   - 根据"文件路径"列的路径读取完整文件
   - 路径格式：`.specify/memory/...` 
   - 必须读取整个文件，不要只读取部分

2. **理解文件内容**
   - 文件包含该技术栈的完整使用说明
   - 包括API方法、参数说明、配置方式、示例代码
   - 根据文件内容设计方案或编写代码

3. **应用到任务**
   - 将文件中的知识应用到当前任务
   - 遵循文件中的规范和最佳实践
   - 使用文件中提供的API和配置方式

### ⚠️ 重要注意事项

1. **命中即读取**：只要关键词命中，无论任务大小都必须读取对应文件
2. **先查后用**：不要凭记忆或猜测，必须先读取文件再使用知识
3. **完整读取**：不要只读取文件的一部分，要读取完整内容
4. **优先级严格**：应用知识和公共知识不能混用，严格遵循优先级

</usage_instructions>
